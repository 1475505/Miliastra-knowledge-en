# Test Play

# I. Function Overview

The Test Play feature can be used for stage testing. Creators can configure Test Play Settings and Test Play Controls to meet their testing needs in different scenarios, enabling rapid validation of stages.

# II. Function Entry Point

System Menu - Test Play

![Image 2](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mhytu59t9u16/e8fb497c-7b34-4892-8c87-0933bbc63a07.png)

# III. Function Restrictions

The Test Play feature cannot be used while a multiplayer Test Play room is active.

# IV. Test Play Verification

After clicking [Test Play], required verification and upload steps run. If a condition isn't met, an error code pops up for review. When all checks pass, the process continues and the [Test Play Settings] window opens.

![Image 3](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mhytu59t9u16/e63f8cd5-f217-4d32-a6b3-26cc759e4c0b.png)

# V. Test Play Flow

The Test Play Flow consists of: Pre-Test Play preparation(upload, verification process) — Cosmetic Display — Loading Interface — Enter Stage — Stage Settlement

_Test Play Player ID_: Click to load configured data from [Stage Settings], then choose which player ID to use for this playtest.

## 2. Cosmetic Display

![Image 8](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mhytu59t9u16/52270eab-823d-492f-8569-5f3b21e2d0b1.png)

_Cosmetic Display Phase_: When disabled, Test Play skips the Cosmetic Display phase and goes directly to the Loading Interface.

![Image 9](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mhytu59t9u16/7ff52f17-aab7-4f5c-8497-a9ce71dfaf02.png)

Players trigger Stage Settlement by completing the challenge or exiting manually

To speed up testing and reduce Test Play steps, turn off [Enable Test Play Settlement] in [Test Play Options] to skip settlement-related steps.

# VI. In-Game Controls for Test Play

## 1. Exit Button

![Image 14](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mhytu59t9u16/fd7db58c-6b70-47df-a9ac-1838276409dd.png)

During playtest, open the [Test Play Controls] panel via the top-left Exit button or the Esc hotkey.

![Image 15](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mhytu59t9u16/e0ab8778-9cd1-434d-b989-02b4dd1733d0.png)

## 2. Test Play Controls

![Image 16](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mhytu59t9u16/1d019d9c-e3e8-401a-a4c2-7f9fca795f3d.png)

### (1) Continue Challenge

Clicking the [Continue Challenge] button will return you to the current playtest session.

![Image 17](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mhytu59t9u16/be1bc897-8a9c-4d60-b5fc-5dc8c14badd8.png)

### (2) Retry Challenge

When the [Retry Challenge] button is clicked, all startup processes from the stage entry sequence are executed again. Once the processes finish, a new playtest run starts.

Note: If an error prevents an immediate restart, exit playtest and start the playtest flow again.

![Image 18](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mhytu59t9u16/9ced4af2-ce55-4cd5-b49d-e3c24fc4c846.png)

### (3) Abandon Challenge

Click to end the current playtest immediately and trigger the corresponding non-standard settlement result.

![Image 19](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mhytu59t9u16/714876bc-9006-4120-8d76-ac6722190b06.png)
