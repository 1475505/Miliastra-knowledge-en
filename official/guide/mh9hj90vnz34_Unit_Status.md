# Unit Status

# 1. Unit Status Component Functions

The Unit Status Component is a component used to configure the default [Unit Status](https://act.hoyoverse.com/ys/ugc/tutorial//detail/mh6rh59iil2i) possessed when an entity is created

For example: Configuring the default [Increase Movement SPD] unit status on a character entity will increase their Movement SPD upon creation; configuring the default [Mount Special Effects] unit status on a mechanism will make it display certain looping effects upon creation

Multiple unit status can take effect simultaneously on the unit status component

# II. Editing Unit Status Components

**Step 1**: Add Unit Status Component

![Image 2](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mh9hj90vnz34/c9025121-28d1-42f3-8239-deb918beb589.png)

1. Switch to the entity or prefab component tab

2. Find or add a unit status component

The unit status component is a default mounted component for all units (i.e., it will be mounted by default when the entity is created). Therefore, for newly created entities or prefabs, you can find the unit status component directly in the Components tab

If it doesn't exist, you can add a unit status component through the Add Common Component button

**Step 2**: Add Unit Status

On the details page of unit status, you can click [Select Status] to add a new unit status configuration

![Image 3](https://act-webstatic.hoyoverse.com/ugc-tutorial/knowledge/sea/en-us/mh9hj90vnz34/6a01d897-1399-4fc4-8ff7-8c78d1ea96f3.png)

In the pop-up tab, you can view some preset unit statuses as well as unit status configured by creators (Craftspeople)
