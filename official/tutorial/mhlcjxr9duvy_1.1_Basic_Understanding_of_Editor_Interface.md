# **In-Game Editor**

### Feature Overview

**In-Game Editor:**The core Genshin Impact Editor is an in-game tool that integrates functionalities including in-game system settings, custom object/monster creation, character/skill/projectile creation, basic value configuration, scene building and object placement, interface creation, and camera control.

![Image 4](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/ca8fcc725c6344d0bce4fe7d7449fdce_1757675605.png)

The in-game editor mainly comprises 7 modules: **Stage Settings**, **Terrain Editing**, **Entity Placement,****Prefab Library**, **Combat Preset**, **UI Controls**, and **Main Camera**. Below is a brief overview of each module's function and usage:

## Stage Settings

### How to Access

Click the upper left corner of the editor ![Image 5](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/f7ac1fac432e43cfa8de9a70db8b9064_1757675605.png) (or press the ESC key) to open the left panel and the ESC menu bar, then click ![Image 6](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/6cc4c12c58cf4d249db9f3b58c527fc4_1757675605.png) to enter

![Image 7](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/9ded9be1c40c41379b683116a7d23eb1_1757675606.png)

### Feature Overview

Stage settings include the following 8 global system settings:

● Basic: Stage environment boundaries, player count limits, and other basic system information;

● Faction: Team tags for players/objects/monsters, friendly/hostile relationships, tags used for default attack logic and condition judgment;

● Spawn Point: Setting the locations where player-controlled characters spawn when entering the game;

● Revive Point: Setting the locations where player-controlled characters respawn after death

● Player Count Settings: Set the number of players for the stage and configure player groups. ● Loading Screen: Graphics Configuration: Set up the visual elements that appear when the stage is loading. ● Settlement: Settlement Logic: Define the rules and conditions for how the stage is settled, including win/loss criteria and scoring.

● Terrain Navigation: Pathfinding Mesh: Set up the navigation mesh for enemies (monsters) to determine their movement paths.

![Image 8](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/5071ce8c9ea041ada86874f3b0eaf5cc_1757675606.png)

_**For more information on stage settings, please refer to [Comprehensive Guide > Stage Settings]_

### Feature Overview

**Miliastra Sandbox:** Tools that operate independently from the Genshin Impact Editor, used for resource indexing, logic programming (node graphs), and debugging.

![Image 37](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/86ce6367f21a4aecb4b265e06f3fd026_1757675602.png)

Miliastra Sandbox mainly include 7 modules: Resource Explorer, Server Node Graph Resource Explorer, Client Node Resource Explorer, Node Explorer, Log, Server Signal Explorer, and Load Detection. The function and usage of each module is briefly outlined below:

## Resource Explorer

### Feature Overview

**Resource Explorer:** Displays resource indexes for "objects placed in the scene" in the current game, "custom prefabs/enemies" in the Prefab Library, and more (cannot be placed);

![Image 38](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/aa19f390e0d54edb81a086e99613ea75_1757675603.gif)

## Server Node Graph Resource Explorer

### Feature Overview

**Server Node Graph Resource Explorer:**It is reponsible for managing node graphs running on the server, creating and managing [Entity Node Graph], [Status Node Graph], and [Class Node Graph], and importing related external node graphs;

![Image 39](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/761cf02577da48c293bee2821c9562cc_1757675604.png)

## Client Node Resource Explorer

### Feature Overview

**Client Node Resource Explorer:**Responsible for managing node graphs running on the client (local), creating and managing [Local Filter Node Graph] and [Skill Node Graph], and importing related external node graphs;

![Image 40](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/9cea3b0222c24f3daebbc37e1231f487_1757675605.png)

## Node Manager

### Feature Overview

**Node Manager:** View current node categories and node details, create and manage composite nodes, and import external composite nodes;

![Image 41](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/1cdf4841b04c44f7b270e403368f3dd4_1757675605.png)

## Log

### Feature Overview

**Log:**Filter specific server node graphs and view their runtime logs while the stage runs (for debugging purposes)

![Image 42](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhuhsdi19rlg/eda82d8da9d149f2b65fc9023506c9bf_1757675605.png)
