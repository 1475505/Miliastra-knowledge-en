# Character Disruptor Device: Apply Force to Characters

## Feature Example 1: Vertical launch after entering area

### 1. Concept Overview

**Character Disruptor Device:**When approaching a certain area, the character will be affected by the set force and change their movement state

**The character disruptor device only affects [Characters]

### 2. Course Content

**Feature:**Create Bouncy Mushrooms that vertically propel characters when they enter specific areas due to the applied force

**Effect Demonstration:**

![Image 2](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/d25cc61becef4c2ba97aba0be5d7e117_1757675177.gif)

### 3. Steps to Follow

**1. [In-Game Editor] Add components**

Create a prefab as a launchable object (using Bouncy Mushrooms as an example in the tutorial), click ![Image 3](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/3628084295a04cf78eaefe84568e81c8_1757675177.png)on the panel to enter the components page, then click ![Image 4](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/11b08166db60465db8893a057101c686_1757675178.png) and select [Character Disruptor Device] to add it.

![Image 5](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/308c62195a2f43038399727e9905f29c_1757675178.png)

**2. [In-Game Editor] Add character disruptor device**

After opening [Components > Character Disruptor Device], there is no character disruptor device by default. Click [+] to add it.

![Image 6](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/5299134531ea4dd3b75b2090e45c7c35_1757675178.png)

**3. [In-Game Editor] Configure panel parameters**

After adding the [character disruptor device], you can adjust the device type, disruption rules, trigger area, and other parameters. To achieve the expected functionality, the configuration in this tutorial is as follows:

Features to Implement Corresponding Configuration Items
Trigger upon entering area Adjust the [Trigger Area] to a sphere shape and configure the appropriate center point and radius

_**Can be freely adjusted based on actual needs during creation_
Launch the character into the air Set [Device Type] to Ejector
Launch direction is vertically upward Set [Disruption Direction] to the specified direction, and set Y-axis direction vector and velocity

**Can be freely adjusted based on actual needs during creation

![Image 7](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/8ee838998a724f96861370cb76bd0714_1757675178.gif)

_**Configuration Item Description:_

_Stable Phase Duration (s): The duration during which the character is continuously affected by the character disruptor device, maintaining the velocity during this period_

_Decay Phase Duration (s): The duration that it takes for the force from the character disruptor device to gradually decreases to 0 after the disruption period ends_

**4. [In-Game Editor] Add components (**Settings here only enhance presentation, not related to actual functionality)**

To make the launch effect appear more animated and noticeable, every time the player enters the collision trigger zone for the launch, the Bouncy Mushroom will also have a squeeze-and-launch animation. Add [Component > Collision Trigger] and adjust it to the appropriate size (matching the character disruptor device's trigger range)

![Image 8](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/e663c7b0bef04644ae89e4a644f944d8_1757675179.png)

**5. [Miliastra Sandbox] Node Graph Configuration (**Settings here only enhance presentation, not related to actual functionality)**

In the entity node graph, add and configure the node graph for this prefab (see recommended settings in the image below). To achieve the desired functionality, the tutorial configuration is as follows:

Features to Implement Corresponding Configurations
Feature 1

When players enter the collision trigger area, the Bouncy Mushrooms will compress and bounce Trigger Scenario Event triggers when player enters the collision area[Event Node]: When entering collision trigger
Condition//
Result 1 Bouncy Mushroom compresses and launches[Execution Node]: Set preset state to change value
Function 2

Bouncy Mushroom recovers when player exits collision trigger area Trigger Scenario Event triggers when player exits collision trigger area[Event Node]: When exiting collision trigger
Condition//
Result Bouncy Mushroom recovers[Execution Node]: Set preset state to initial value

![Image 9](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/28223fca0599406ebbbd24b965a51ec3_1757675179.png)

**6. [In-Game Editor] Mounting node graph**

Mount the node graph on the prefab and save the prefab settings. Place the prefab in the scene and enter Test Mode to try out the feature.

![Image 10](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/16484a59355940d2a41540379c45e9b5_1757675179.gif)

## Feature Example 2: Perform Traction and Teleportation When Players of Specific Factions Enter an Area

**Feature Description:**Create a character disruptor device (teleporter) that only affects specific factions

**Effect Demonstration:**

_**For demonstration purposes, the tutorial has added UI display of the character's current faction (the method will not be explained in detail here)._

****Text Display in Demos:**The ability to customize text display is unlocked after reaching a specific Craftsperson Level. In the demo, the focus is primarily on demonstrating the effects of this feature.

![Image 11](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/5df88c41b4fc4bda8e516225291c24c1_1757675174.gif)

**Implementation Steps:**

1. Create a prefab as an interactive object, add [Components > Character Disruptor Device], adjust the device type, disruption rules, trigger area, and other parameters. To achieve the expected functionality, the configuration in this tutorial is as follows:

Features to Implement Corresponding Configuration Items
Trigger upon entering area Adjust the [Trigger Area] to a cube shape and configure the appropriate center point and scale

**Can be freely adjusted based on actual needs during creation
Pull and teleport character to a specific area Set [Device Type] to Traction Device
Traction direction is X-axis Set [Disruption Direction] to the specified direction, and set X-axis direction vector and velocity

**Can be freely adjusted based on actual needs during creation
Traction can be interrupted Enable the [Jump Interrupt] option

![Image 12](https://act-webstatic.hoyoverse.com/ugc-tutorial/course/sea/en-us/mhlzu3rolkr2/9baff45844ed4ddbb0ebe31c644f32f1_1757675175.gif)

2. [Client Node Graph > Filter Node Graph] Configure the faction filtering effect. To achieve the expected functionality, the tutorial configuration is as follows:

Features to Implement Corresponding Configuration Items
Feature

Players can be teleported when they belong to Faction 1. This is disabled when they belong to other factions Trigger Target Query the faction of the entity entering the character disruptor device trigger area[Target Entity] = The entity entering the character disruptor device trigger area

_**This entity is a character, switch to player before querying faction_
Condition The player entering the character disruptor device trigger area belongs to Faction 1 Operation Node: [Equals] Check if target entity's faction is Faction 1

_**Remember to check if the variable type is [Faction]._
Result//
